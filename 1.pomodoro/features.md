# ポモドーロタイマー - 機能要件一覧

## 📌 機能分類

```
【フロントエンド機能】
├── タイマー表示・制御
├── UI/UX インタラクション
├── 進捗表示・可視化
├── ローカルデータ管理
└── ユーザーフィードバック

【バックエンド機能】
├── セッション履歴管理
├── 統計データ API
└── サーバー側永続化

【共通機能】
├── タイマーコアロジック
├── ストレージ抽象化
└── テスト機能
```

## 3️⃣ ビジネスロジック機能（timer_logic.py）

### タイマー計算ロジック

| # | 機能 | 詳細 | 関数名 | 優先度 |
|---|------|------|--------|--------|
| L-1.1 | セッション初期化 | 作業時間・休憩時間を初期設定 | `PomodoroSession.__init__()` | **必須** |
| L-1.2 | 経過時間カウント | 1秒ごとに経過秒数を加算 | `tick()` | **必須** |
| L-1.3 | 残り時間計算 | 残り秒数を計算（副作用なし） | `get_remaining()` | **必須** |
| L-1.4 | 完了判定 | セッション完了判定 | `is_complete()` | **必須** |
| L-1.5 | 進捗率計算 | 0〜1の小数で進捗率を返す | `get_progress_ratio()` | **必須** |
| L-1.6 | リセット | 経過時間を0に戻す | `reset()` | **必須** |
| L-1.7 | 時間フォーマット | 秒を "MM:SS" 形式に変換 | `format_time()` | **必須** |

### 統計追跡ロジック

| # | 機能 | 詳細 | 関数名 | 優先度 |
|---|------|------|--------|--------|
| L-2.1 | セッション記録 | 完了したセッションを追跡 | `WorkSessionTracker.add_completed_session()` | **必須** |
| L-2.2 | 統計集計 | 完了数・累計時間を集計 | `WorkSessionTracker.get_stats_dict()` | **必須** |
| L-2.3 | 日付判定 | タイムスタンプから同日判定 | `is_same_day()` | **必須** |
| L-2.4 | 統計リセット | 日付変更時に統計をリセット | `reset_if_new_day()` | **必須** |

## 📝 備考

- 各機能は「優先度」に基づいて実装順序を決定してください
- **必須機能**が完成した時点で、ユーザーが使用可能なアプリケーションになります
- **中優先度**はユーザーフィードバックに基づいて段階的に追加
- **低優先度**はニーズに応じてオプション実装
- 各フェーズのテスト完了後、次のフェーズに進むこと
